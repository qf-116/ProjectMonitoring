## 1. 执行摘要

### 1.1 问题陈述

公司变压器项目同时包含非标设备与批量产品订单。一个合同项目可能包含多种产品、多台交付，生产、初检、成品入库、销售出库等状态会同时存在。现有数据分散在 CRM、PLM、MES、QMS、WMS 等系统中，管理层缺少统一视图判断：

- 最近需要交付的项目兑现情况如何；
- 最近签订或立项的项目推进是否健康；
- 哪些项目需要关注，卡在设计、生产、质量、入库还是发货；
- 从指标能否下钻到项目和产品明细。

### 1.2 解决方案

大屏采用两个一级总览页面承载两类管理问题：

1. **项目交付**：默认入口，面向“最近应交付项目”的交付兑现、临期/逾期、待发货和质量阻塞。
2. **项目进展**：面向“最近签订 / 立项项目”的启动、BOM 发布、生产推进和前期阻塞。

同时保留：

- **项目查询**：全库查询、筛选、重点关注/取消关注；
- **项目详情**：单合同全链路、产品明细、里程碑时间轴；
- **产品业务摘要抽屉**：单产品跨系统摘要；
- **自定义里程碑维护**：允许人工维护系统无法自动获取的项目关键事件，并可选择纳入总览链路。

### 1.3 成功标准

- **数据时效**：页面数据延迟 ≤ 5 分钟，页面显示“数据更新于 N 分钟前”。
- **口径一致性**：同一指标在指标卡、链路、查询、详情中数值一致，可勾稽。
- **下钻闭环**：风险指标或风险清单 ≤ 2 次点击定位到具体合同项目。
- **管理有效性**：领导能在首屏区分“本期交付兑现风险”和“新签项目推进风险”。
- **可读性**：1920×1080 大屏下信息密度充足，避免大量小卡片割裂；状态不单纯依赖颜色。

## 2. 用户角色

| 角色 | 场景 | 核心诉求 |
|------|------|----------|
| 高管 / 决策层 | 例会扫屏、办公室查看 | 快速判断交付是否有风险、哪些项目需要协调 |

## 3. 全局信息架构

四个视图 + 一个抽屉：

1. **项目交付**（delivery）——默认入口，查看本期应交项目的交付兑现。
2. **项目进展**（progress）——查看最近签订 / 立项项目的推进健康度。
3. **项目查询**（query）——全库查询、多维筛选、重点关注/取消关注。
4. **项目详情**（detail）——单合同全链路、产品明细、项目里程碑。
5. **产品业务摘要抽屉**（drawer）——单产品跨系统摘要。

**顶部导航：** 项目交付 / 项目进展 / 项目查询。

**默认页面：** 项目交付。

**下钻规则：** 项目详情为项目查询及风险清单的下钻页面，不作为平级导航；进入详情时顶部高亮“项目查询”，通过“← 返回项目查询”回退。

## 4. 页面需求

### 4.1 项目交付页面

#### 页面定位

回答“最近需要交付的项目，目前能不能按期交付”。关注交付兑现、临期、逾期、待发货、质量/生产阻塞。

#### 时间口径

- 默认：本季度。
- 筛选项：本月 / 本季度 / 本年 / 近 30 天 / 自定义。
- 统计口径：按计划交付日期 / 合同交付日期落入所选期间统计。

#### 首屏指标

| 指标 | 口径 | 类型 |
|------|------|------|
| 本期应交项目数 | 计划交付日期落入期间的合同项目数 | 期间 |
| 本期应交台数 | 本期应交合同产品明细数量汇总 | 期间 |
| 已销售出库台数 | 期间内有效销售出库数量 | 期间 |
| 本期交付率 | 已销售出库台数 / 本期应交台数 | 期间 |
| 本期交付缺口 | 本期应交台数 - 已销售出库台数 | 期间 |
| 临期风险 | 未来 15 天到期且未完成发货的剩余台数 / 项目数 | 实时 |
| 已逾期未交付 | 已过交付日期且未完成发货的剩余台数 / 项目数 | 实时 |
| 成品待发货 | 成品入库 - 销售出库 | 实时 |

#### 交付全链路

链路节点：

```text
本期应交 → 生产完成 → 初检完成 → 初检合格 → 成品入库 → 销售出库
```

节点差额：

- 生产缺口 = 本期应交 - 生产完成；
- 初检积压 = 生产完成 - 初检完成；
- 待入库 = 初检合格 - 成品入库；
- 待发货 = 成品入库 - 销售出库。

交付页面可展示影响本期交付的质量、生产、待发货阻塞；BOM 阻塞不作为交付页主指标，除非明确影响本期应交。

#### 趋势图

展示：本期应交 / 生产完成 / 销售出库。

要求：

- Y 轴标明台数量级；
- X 轴跟随所选时间段；
- 图例标明单位；
- hover 可查看该时点明细。

#### 底部区域

1. **重点关注项目**
   - 展示人工标记的重点项目；
   - 交付页优先排序：逾期、临期、待发货、质量阻塞；
   - 采用轮播，避免重点项目数量增加后纵向过长；
   - 查询列表和详情页提供“设为重点关注 / 取消关注”。

2. **风险关注清单**
   - 页签：交付风险、质量 / 生产阻塞、停滞待发货、数据异常；
   - 每行展示合同号、客户、风险原因、当前节点、责任部门、影响数量、最近进展；
   - 每行可点击进入项目详情；
   - 合同号必须能真实匹配项目，匹配失败须提示，不允许假下钻。

### 4.2 项目进展页面

#### 页面定位

回答“最近签订 / 立项的项目，当前推进是否正常”。关注启动、BOM 发布、生产推进、前期阻塞。

#### 时间口径

- 默认：本季度。
- V1 默认使用合同签订日期。
- 后续如系统具备项目立项日期，可切换为项目立项日期或增加口径选项。

#### 首屏指标

| 指标 | 口径 | 类型 |
|------|------|------|
| 本期新签 / 立项项目数 | 合同签订日期或立项日期落入期间的项目数 | 期间 |
| 本期新签应交台数 | 本期新签项目的应交台数汇总 | 期间 |
| 合同产品种类数 | 合同号 + 产品编码 + 型号规格去重 | 期间 |
| BOM 发布完成率 | 已发布 BOM 产品种类 / 合同产品种类 | 实时 |
| 已有生产进展项目数 | 存在生产完成或产成品入库记录的项目数 | 实时 |
| 生产完成台数 | 本期新签项目当前累计生产完成台数 | 实时 |
| 签约后无进展项目 | 超过阈值仍无 BOM 或生产进展的项目数 | 实时 |
| 当前阻塞项目数 | BOM、设计变更、生产停滞、数据异常等阻塞项目数 | 实时 |

> 注意：“已有生产进展项目数”比“已进入生产项目数”更稳妥，因为当前确认的 MES 可靠节点是工单产成品完成入库，不一定有统一生产开工事件。

#### 推进全链路

链路节点：

```text
合同签订 / 立项 → BOM 发布 → 生产进展 → 生产完成 → 初检完成 → 初检合格 → 成品入库 → 销售出库
```

原型首屏可压缩展示为：

```text
合同产品 → BOM发布 → 生产进展 → 生产完成 → 初检完成 → 成品入库
```

说明：项目进展页链路允许按节点业务口径展示，不强制使用同一单位。合同产品、BOM 发布适合按“种”统计；生产进展适合按“项目”统计；生产完成、初检、入库适合按“台”统计。界面必须在面板说明中标注，避免误解为同一分母漏斗。

节点差额：

- BOM 缺口 = 合同产品种类 - BOM 已发布种类；
- 未形成生产进展 = 新签项目数 - 已有生产进展项目数；
- 生产缺口 = 新签应交台数 - 生产完成台数；
- 初检积压 = 生产完成台数 - 初检完成台数；
- 待入库 = 初检合格台数 - 成品入库台数。

#### 趋势图

展示：新签应交 / BOM 发布 / 生产进展。

#### 底部区域

1. **重点关注项目**
   - 进展页优先排序：无进展、BOM 阻塞、设计变更、生产停滞；
   - 展示合同号、客户、应交数量、BOM 发布情况、当前推进卡点、责任部门。

2. **推进风险清单**
   - 页签：启动滞后、BOM / 设计阻塞、生产停滞、数据异常；
   - 展示无进展项目、BOM 阻塞种类、生产停滞项目、设计变更项目、数据异常项目。

### 4.3 项目查询页面

#### 功能

- 按合同号、客户、产品编码、型号规格、项目状态、关注情况、BOM 状态筛选；
- 快捷页签：全部 / 在执行 / 已完成 / 有异常；
- 查询结果分页展示；
- 支持重点关注 / 取消关注操作；
- 点击合同进入项目详情；
- 顶部合并展示项目状态结构和数据更新时间。

#### 项目状态结构

不使用占空间的环图，改为一行“类型 + 数字”：

```text
执行中 N · 已完成 N · 未开始 N · 暂停/取消 N · 共 N 个在库项目 · 数据更新于 N 分钟前
```

#### 表格列

合同号、客户、产品种类、BOM 发布、应交、生产完成、初检完成、初检合格、不合格、成品入库、已发货、发货完成率、项目状态、关注标签、最近进展、操作。

### 4.4 项目详情页面

#### 功能

- 合同摘要：签订日期、交付日期、产品种类、当前有效应交、BOM 发布、一次检验合格率、最近进展、责任部门；
- 发货完成率环图：已发货、剩余、状态标签；
- 全链路进展：应交 / 生产 / 初检 / 合格 / 入库 / 出库；
- 产品明细表：按产品展示进度、BOM、质量、入库、发货；
- 项目里程碑时间轴：系统自动里程碑 + 自定义里程碑；
- 维护自定义里程碑；
- 设置或取消重点关注。

### 4.5 产品业务摘要抽屉

点击项目详情中的产品行后打开抽屉，展示：

1. **设计与物料**：BOM 状态、版本、发布时间、采购入库单、来料检验情况；
2. **生产与质量**：应交、生产完成、初检完成、初检合格、不合格、返修；
3. **入库与发货**：成品入库、销售出库、未发货、发货率、最近生产和出库时间。

### 4.6 自定义里程碑

#### 背景

合同、BOM 发布、生产完成、初检完成、成品入库、销售出库等是系统自动里程碑。现场勘察、设计变更、协调会、物流延误等系统外事件，需要人工维护。

#### 要求

- 项目详情中展示里程碑时间轴；
- 系统里程碑与自定义里程碑混排，按时间倒序；
- 自定义里程碑可选择“纳入总览全链路里程碑统计”；
- 如果纳入总览，应作为项目全链路的一部分展示，不单独做“重要事件动态”；
- 自定义里程碑维护是本大屏唯一写入口，需权限控制和审计。

#### 字段

| 字段 | 含义 | 说明 |
|------|------|------|
| id | 里程碑 ID | 系统生成 |
| contractNo | 合同号 | 关联项目 |
| time | 里程碑时间 | 业务发生时间 |
| type | 类型 | 里程碑 / 异常风险 / 协调决策 / 设计变更 / 其他 |
| title | 名称 | 建议 ≤ 40 字 |
| desc | 说明 | 建议 ≤ 500 字 |
| owner | 维护人 / 来源 | 系统应取登录账号 |
| source | 来源 | system / manual |
| showInOverview | 是否纳入总览链路 | 默认 false |
| createdBy / createdAt | 创建审计 | 后端记录 |
| updatedBy / updatedAt | 修改审计 | 后端记录 |

## 5. 数据口径规格

### 5.1 关键关联键

跨系统关联优先使用：

```text
合同号 + 产品编码 + 型号规格
```

说明：合同号只能稳定关联到项目，但一个合同可能包含多个产品，因此不能只用合同号做产品级关联。合同明细行 ID 可同步到 PLM、MES、QMS、WMS 时优先使用合同明细行 ID；否则使用上述三字段匹配。

### 5.2 系统节点定义

| 节点 | 来源 | 有效状态 |
|------|------|----------|
| 合同签订 | CRM | 合同生效 / 签订完成 |
| BOM 发布 | PLM | BOM 发布状态为已发布 |
| 生产完成 | MES | 工单产成品完成入库 |
| 初检完成 / 初检合格 | QMS | 取初次检验结果；QMS 有初次检验、返修检验标识 |
| 采购入库 | WMS / ERP | 采购入库单有效 |
| 成品入库 | WMS | 生产入库单有效 |
| 销售出库 | WMS | 销售出库单有效；项目完成标准为发货完成 |

### 5.3 存量与流量

- **期间指标（流量）**：一段时间内发生的数量，随时间筛选变化，如本期应交、本期出库、本期交付率、新签项目数。
- **实时指标（存量）**：当前状态快照，不随时间筛选变化，如临期风险、逾期未交付、成品待发货、BOM 发布率、当前阻塞项目。

界面必须用“期间 / 实时”徽标明确区分，避免口径穿帮。

### 5.4 风险规则

| 风险 | 判定规则 |
|------|----------|
| 临期风险 | 未完成发货，交付日期在未来 0–15 天内 |
| 已逾期 | 未完成发货，交付日期早于当前日期 |
| 停滞待发货 | 成品已入库但销售出库未完成，超过等待阈值 |
| 初检异常 | 初次检验存在不合格数量 |
| BOM 阻塞 | 合同产品 BOM 未发布，影响后续生产或交付 |
| 启动滞后 | 签约后超过阈值无 BOM 或生产进展 |
| 生产停滞 | 最近生产进展超过阈值无更新 |
| 数据异常 | 合同号 + 产品编码 + 型号规格无法匹配，或跨系统数量不一致 |

## 6. 技术规格

### 6.1 架构

```text
CRM / PLM / MES / QMS / WMS / ERP
        ↓
数据抽取 / 接口拉取 / 定时聚合
        ↓
统一口径计算层
        ↓
只读聚合 API
        ↓
大屏前端
```

本阶段仅开发前端说明原型，不开发后端服务。真实开发时后端负责统一口径计算，前端不做跨系统业务推导。

### 6.2 前端要求

- 原型为单文件 `index.html`；
- 目标分辨率 1920×1080；
- 页面切换 ≤ 300ms；
- 支持分钟级数据刷新展示；
- 所有接口文本需 HTML 转义；
- 状态色需配合文字标签，不能只靠颜色表达。

### 6.3 视觉要求

- 左上角展示公司 Logo：希望智能 HOPE INTELLIGENT，透明背景，不加白色底板；
- 深色工业驾驶舱风格，克制、专业，减少霓虹和装饰；
- 顶部指标使用连续式指标带，减少小方块割裂；
- 链路差额、风险汇总、重点关注汇总使用轻分隔信息条；
- 重点关注标识使用“重点”文字，不使用星标或 emoji；
- 页面标题、按钮、标签、状态标识统一字号、圆角和颜色层级。

## 7. 非目标

本期不做：

- 金额、成本、回款等财务指标；
- 移动端适配；
- 督办、导出、审批流；
- 多语言；
- WebSocket 实时推送；
- 返修全过程工艺追溯；
- 对 CRM / PLM / MES / QMS / WMS 的业务数据写操作。

唯一例外：自定义里程碑维护，但真实开发必须有权限与审计。

## 8. MVP 范围

| 模块 | 范围 |
|------|------|
| 项目交付 | 默认入口；交付指标、交付链路、交付趋势、交付风险清单 |
| 项目进展 | 新签 / 立项项目推进指标、推进链路、推进趋势、推进风险清单 |
| 项目查询 | 全库查询、筛选、状态摘要、重点关注 / 取消关注 |
| 项目详情 | 合同摘要、全链路、产品明细、里程碑时间轴 |
| 产品抽屉 | 单产品跨系统业务摘要 |
| 里程碑 | 系统里程碑展示、自定义里程碑维护、可纳入总览链路 |
| 视觉规范 | Logo、深色工业风、连续指标带、风险标签体系 |

## 9. 主要风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| 合同产品无法跨系统可靠匹配 | 项目/产品进度对不上 | 优先同步合同明细行 ID；否则使用合同号 + 产品编码 + 型号规格，并做数据异常清单 |
| 口径不一致 | 管理层不信任大屏 | 以本 PRD 第 5 节为唯一口径；上线前做抽样勾稽 |
| 生产进展节点定义不统一 | 项目进展误判 | V1 使用 MES 工单产成品完成入库作为可靠生产完成节点；生产启动类指标谨慎命名为“已有生产进展” |
| 采购接口暂不可用 | 物料过程不可见 | 本期不展示齐套率；只在到货后通过采购入库 / 来料检验体现 |
| 重点关注项目越积越多 | 总览区过长 | 采用轮播，并在查询与详情提供取消关注 |
| 自定义里程碑滥用 | 总览链路失真 | 默认不纳入总览，需人工勾选；真实开发需权限、审计和必要审核 |

## 10. 验收总纲

上线或原型评审需同时满足：

1. 顶部导航存在“项目交付 / 项目进展 / 项目查询”，默认打开项目交付。
2. 项目交付与项目进展指标不混用：交付页看应交与兑现，进展页看新签与推进。
3. 时间口径明确：交付按计划交付日期，进展 V1 按合同签订日期。
4. 风险清单可下钻到真实项目详情，不能假跳转。
5. 项目查询支持重点关注 / 取消关注。
6. 自定义里程碑如需显示到总览，必须纳入全链路节点语义，而不是单独“事件动态”。
7. 页面视觉符合希望智能品牌与深色工业驾驶舱规范。
